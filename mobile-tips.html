Tips and Tricks for Yesware's mobile application


The Problem

Yesware’s mobile app was quickly gaining traction, and at the time, there was no good way to easily convey best practices. The Mobile team needed an FAQ-style document that quickly answered the most common support questions. The feature offerings also differed depending on the plan a user was on, so the FAQs presented also needed to be relevant to the user's plan.

My Role

As the Designer for the Mobile Tips & Tricks project, I was tasked with designing and implementing a responsive web page to be hosted on our Marketing site. In terms of task breakdown, I was responsible or creating low-fidelity wireframes (15%), creating high-fidelity wireframes (35%), and implementing the designs using HTML, CSS, and jQuery (50%).

Toolbox

Sketch, Google Drive, Invision, Sublime Text 3, HTML, CSS, jQuery

Project Scope

This was a relatively small project to be completed in 2 weeks. I budgeted 1 week for designing the pages, gathering feedback, and incorporating the changes into the subsequent versions. The second week would be reserved for implementing the designs using HTML, CSS, and jQuery and performing some QA to make sure the site rendered properly across different platforms.
